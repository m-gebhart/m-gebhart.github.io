Gigantic: Rampage Edition

2024 · 5v5 MOBA Hero Shooter · Commercial Project (1+ years) · Unreal Engine 3 / C++ / UnrealScript / ActionScript / Scaleform / Flash

During my internship and Jr. programming role at Abstraction Games, I had the opportunity to work on the re-release of Gigantic (original release by Motiga in 2017) where I was part of the core code team from March 2023 until July 2024. Released as Gigantic: Rampage Edition in April 2024, Abstraction provided many improvements and additions such as entire UI overhauls, more guided tutorials, a new game mode called Rush, two new maps, and two new heroes, namely Roland and Kajir.

My responsibilities mostly consisted of UI programming to increase accessibility and usability despite the technical debt warranted by Unreal Engine 3 and Flash. Later, I also worked on other game programming aspects to fix and create combat and attack behaviour for single heroes.

UI Programming

Footer Action Bar:

  • introducing support for dynamic animations, i.e., a ring-like "filling-up" animation around button images to indicate held buttons of in-progress actions (e.g., purchase actions).
  • ensuring consistent spacing and enumerations of elements, including:
    • loading in button images either next to or in-between text characters. This is also dependent on the chosen language or input type (gamepad / mouse).
    • a priority system that keeps a consistent position index of certain actions (e.g., "ESC" or "Back" always first) independent from data sheets definitions.
    • contextual positioning that relatively moves the footer bar horizontally. This is based on distances to other near objects, e.g., to prevent overlaps with party member block in bottom corner.

Character Builds:

  • gamepad navigation support.
  • custom controls by introducing a drag'n'drop system to better manage order of a hero character's upgrade build:
    • detect targeted items of current drag by using a two-pointer system (i.e, find closest item to current mouse coordinates) for placement and hover feedback.
    • instant feedback what combinations and modifications are possible while editing:
      • indications that some upgrades are dependent on other base upgrades and might require a certain order.
      • index order update when modifying already equipped upgrades, e.g., only swap valid combinations, or "move up" upgrades after deleting upgrades with lower indices to prevent gaps.

Voice Chat UI:

  • unified update system to the voice chat UI based on delegates and events:
    • custom voice UI for the party block, hero selection screen and scoreboard.
    • general UI solutions for gameplay, transition screens and other.
  • support for mute and block actions as well as joining or leaving players to refresh session states.

Shield and Health UI:

  • complete rework of shield and health UI to now share the total bar instead of one bar overlapping the other:
    • smoothened shield bar's refill animation when recharging.
    • improved shield bar's damage loss "burn" animation.
    • extra health buff (prior display as separate bar) now part as usual health.
  • generalized shield bar implementation to improve display for any heroes, no matter whether shield is permanent or temporary only:
    • e.g., whilst Rutger has a permament shield that auto-heals, the shield could also only be temporary with HK's or Zandora's skill.

End of Match:

  • Rush mode: with the option to change heroes mid-match, I added an internal counting system to be able display the most played hero by each player.
  • Finalized MPR system:

And many other UI implementions:

  • skill interval counter to the HUD, showing when a certain skill can execute a timed effect, e.g., Wu can do a Cyclone Kick every 3s on its primary attack.
  • rework and visual overhaul of the hero selection screen
  • general HUD change to a more blue-green and gradient-based palette for contrast.
  • rework of AoE outlines with contrast colors and limiting visibility to only affected players to better perceive heal and damage outlines of hero skills.
  • fixes to tutorial hint displays, gamepad navigations, and many other UI bugfixes!
